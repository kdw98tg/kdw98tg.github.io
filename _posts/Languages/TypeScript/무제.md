# 화면 상세 설계서: Solve It (UI Screen Specification)

  

이 문서는 'Solve It' 데모 플랫폼의 화면별 UI/UX 컴포넌트, 기능 흐름, 그리고 예외 처리를 명시한 상세 화면 설계서입니다. 각 화면의 기능 및 화면 구성 설계에 집중하였습니다.

  

---

## 1. 랜딩 페이지 (landing.html)

  

*   **화면 목적**: 서비스의 핵심 가치와 비즈니스 모델을 효과적으로 전달하고, 사용자(제공자 및 개발자)의 서비스 진입을 유도합니다.

  

### 1.1. 주요 UI 컴포넌트

1.  **GNB (글로벌 내비게이션 바)**

    *   `로고`: 서비스명 `SolveIt` 노출. 클릭 시 랜딩 페이지 새로고침.

    *   `테마 토글 버튼`: 다크 / 라이트 모드 전환 버튼.

    *   `피드 보기 버튼`: 클릭 시 메인 피드(index.html)로 이동.

    *   `시작하기 버튼`: 브랜드 컬러 오렌지가 가미된 채워진 버튼. 클릭 시 메인 피드(index.html)로 이동.

2.  **Hero 섹션 (메인 배너)**

    *   `아이캐치 태그`: '아이디어가 세상을 바꿉니다' 문구 노출.

    *   `메인 헤드라인`: "불편함이 서비스가 되는 곳"을 강조하는 굵은 텍스트 및 그라데이션 타이틀.

    *   `서브 타이틀`: 서비스 개념 설명.

    *   `CTA 버튼 그룹`:

        *   `아이디어 보러가기` (브랜드 컬러 강조형)

        *   `둘러보기` (아웃라인형)

3.  **플랫폼 통계 섹션 (Stats)**

    *   `공유된 아이디어 수` (2,847개), `참여 개발자 수` (384명), `출시된 서비스 수` (91개)를 보여주는 심플한 수치 카드와 수직 디바이더.

4.  **서비스 프로세스 소개 섹션 (Features)**

    *   3단계 카드형 레이아웃:

        *   `1단계 (불편함을 공유하세요)`: 일상 불편 제안.

        *   `2단계 (공감을 모으세요)`: 사용자들의 공감 및 투표 유도.

        *   `3단계 (서비스로 태어납니다)`: 개발자의 실제 구현과 원작 크레딧 영구 부여 시스템 설명.

5.  **하단 락인 배너 (Footer CTA)**

    *   "지금 바로 시작해보세요" 문구와 함께 메인 피드 유입을 위한 최종 CTA 버튼 배치.

  

### 1.2. 화면 흐름 및 인터랙션

*   **네비게이션 백그라운드 블러**: 사용자가 화면을 아래로 20px 이상 스크롤할 시, 투명하던 GNB가 불투명하게 변하고(rgba(10,10,10,0.85) / backdrop-filter: blur(16px)), 하단 경계선(border-bottom)이 부드럽게 나타납니다.

*   **다이내믹 테마 전환**: GNB 내 테마 토글 버튼을 누르면 전체 CSS 변수(--bg, --text 등)가 반전되며, 로컬스토리지에 저장되어 다른 화면 진입 시에도 유지됩니다.

## 2. 메인 아이디어 피드 페이지 (index.html - Feed Tab)

  

*   **화면 목적**: 등록된 불편함 아이디어를 브라우징하고, 공감 투표를 하거나 상세 페이지로 이동하며, 새로운 아이디어 발의를 탐색하는 메인 홈 화면입니다.

  

### 2.1. 주요 UI 컴포넌트

1.  **GNB (메인 전용 GNB)**

    *   중앙 `페이지 전환 탭`: `아이디어 피드` (활성화 상태) / `개발자 쇼케이스`.

    *   우측 `역할 스위처`: `제공자` / `수요자` 토글 버튼 그룹.

    *   우측 `인증 버튼 그룹`: `로그인` 및 `가입하기`.

2.  **카테고리 & 정렬 바**

    *   `카테고리 버튼 그룹`: 전체, 앱, 웹, AI, 게임, 생활, 소셜 탭 필터.

    *   `정렬 셀렉트 박스`: `인기순`, `최신순`, `댓글순`.

3.  **간이 글쓰기 바 (write-bar)**

    *   유저 프로필 프레임 및 인풋 텍스트 자리의 플레이스홀더 버튼 ("불편한 거 있으세요? 아이디어를 공유해보세요...").

    *   `올리기` 버튼.

4.  **아이디어 피드 카드 리스트**

    *   `아이디어 카드`:

        *   **좌측**: 공감 투표 컨트롤러 (상향 화살표 버튼 + 공감 수 텍스트).

        *   **우측 상단**: 카테고리 배지, `구현됨` 배지 (해당 아이디어가 실제 서비스화된 경우 노출).

        *   **우측 중앙**: 아이디어 제목 (1.4줄 간격) 및 본문 내용 미리보기 (말줄임표 최대 2줄 처리).

        *   **우측 하단**: 해시태그 목록, 댓글 수, 등록 시간 정보, 제안자 아바타 정보.

5.  **사이드바 영역**

    *   `이번 주 인기 아이디어 목록`: 1~5위까지 공감 수 기준의 콤팩트한 텍스트 랭킹.

    *   `활발한 개발자 목록`: 개발자의 프로필, 아이디, 획득 등급 배지(1등급, 2등급, 3등급 등).

    *   `아이디어 공유 유도 CTA 카드`: 설명 텍스트와 함께 즉각 작성 페이지로 연결해 주는 버튼 탑재.

  

### 2.2. 화면 흐름 및 인터랙션

*   **역할에 따른 분기**:

    *   `제공자(Provider)` 모드인 경우: 피드 상단의 간이 글쓰기 바(write-bar)가 보입니다.

    *   `수요자(Seeker)` 모드로 전환 시: 피드 상단의 간이 글쓰기 바가 서서히 사라져 스크롤을 내리지 않고도 더 많은 검증 리스트를 볼 수 있게 배치합니다.

*   **공감 투표**: 카드 좌측의 상향 화살표 버튼 클릭 시 전체 카드를 새로고침하지 않고, 해당 카드의 공감 카운트만 비동기(정수 업데이트 및 토스트 메시지)로 반영됩니다. 이미 투표한 카드는 버튼 색상이 브랜드 오렌지 컬러로 변합니다.

*   **상세 페이지 전환**: 카드의 투표 버튼 영역을 제외한 본문 영역 중 어느 곳이든 클릭하면 해당 아이디어의 상세 페이지(post.html?id=ID)로 자연스럽게 이동합니다.

*   **카테고리/정렬 즉시 필터링**: 카테고리 탭이나 셀렉트 박스 값을 변경하면 피드 영역이 페이드아웃 후 필터링된 카드로 동적 재구성됩니다.

  

### 2.3. 예외 처리 & 마이크로 UX

*   **피드 비어있음 케이스**: 선택한 카테고리에 조건과 맞는 제안이 하나도 없을 경우, "아이디어가 없어요. 첫 번째로 올려보세요!" 안내문이 피드 리스트 중앙에 크게 노출됩니다.

*   **반응형 스크린 최적화**: 태블릿/모바일(1024px 이하) 환경에서는 우측 사이드바가 리스트 아래쪽으로 떨어지거나 사라져 1열 카드 레이아웃으로 변경됩니다.

## 3. 아이디어 상세 페이지 (post.html)

  

*   **화면 목적**: 발의된 아이디어의 전문을 읽고, 다른 유저가 남긴 피드백(댓글)을 보거나 직접 작성하고, 실제 구현된 결과물을 확인하는 상세 소통 창구입니다.

  

### 3.1. 주요 UI 컴포넌트

1.  **뒤로가기 버튼**: 상단에 배치된 "← 피드로 돌아가기" 텍스트 링크. 클릭 시 메인 피드로 복귀.

2.  **메인 상세 카드**

    *   `투표 바`: 상단에 위치. 공감 버튼과 함께 "N명이 공감했어요" 수치 표기.

    *   `게시글 헤더`: 카테고리 배지, `구현됨` 배지, 아이디어 풀 제목, 작성자 프로필(닉네임 + 등록 시간).

    *   `게시글 본문`: 줄바꿈이 정상 보존된 마크업 형태의 상세 줄글(텍스트 줄 간격 1.85로 쾌적한 가독성 제공).

    *   `태그 목록`: 개별 회색 둥근 배지 형태의 해시태그 배치.

    *   `액션 버튼 그룹`: `댓글 N개`, `공유`, `저장` 버튼.

3.  **구현 연계 배너 (impl-block)**

    *   아이디어가 실제로 서비스로 개발되었을 때만 나타나는 그린 라이트 컬러 배너.

    *   서비스 아이콘 정보, "구현됨: 이 아이디어로 만들어진 서비스 [이름]" 타이틀, 개발자 닉네임 및 획득 추천수 표기.

4.  **댓글 영역 (Comments)**

    *   `댓글 개수 배지`: 댓글 등록에 따라 실시간 개수 증가.

    *   `댓글 작성 폼`: 로그인 아바타, 3줄 높이의 텍스트 입력창, `댓글 등록` 버튼.

    *   `댓글 목록`: 개별 유저 아바타, 작성자 닉네임, 작성 시간, 내용이 담긴 타임라인형 리스트.

  

### 3.2. 화면 흐름 및 인터랙션

*   **링크 공유 기능**: `공유` 버튼 클릭 시, 현재 접속 중인 고유 URL이 복사되며 우측 하단에 "링크가 복사됐어요!"라는 초록색 토스트 알림이 팝업됩니다.

*   **댓글 실시간 삽입 및 스크롤 포커스**: 댓글을 입력하고 등록 버튼(혹은 Ctrl+Enter)을 누르면 목록 하단에 즉시 DOM이 동적으로 생성되고, 사용자가 방금 작성한 댓글의 위치로 화면이 부드럽게 스크롤링(scrollIntoView({ behavior: 'smooth' }))되어 즉각 피드백을 전달합니다.

  

### 3.3. 예외 처리 & 마이크로 UX

*   **404 페이지 처리**: 잘못된 ID(쿼리 스트링)로 접근하거나 글이 존재하지 않는 경우, 본문 영역 대신 "게시글을 찾을 수 없습니다"라는 정중한 안내 및 `피드로 돌아가기` 버튼만 출력되도록 예외 렌더링을 진행합니다.

## 4. 아이디어 작성 페이지 (write.html)

  

*   **화면 목적**: 제공자 모드의 핵심 화면으로, 제목, 카테고리, 내용, 태그를 체계적으로 입력하여 검증 대기 상태인 아이디어를 발행합니다.

  

### 4.1. 주요 UI 컴포넌트

1.  **헤더 영역**: 뒤로가기 버튼, 작성을 나타내는 아이콘 이미지, 타이틀 및 친절한 안내 메시지.

2.  **작성 폼 카드 (좌측)**

    *   `제목 입력창`: 플레이스홀더 제안 가이드가 들어간 인풋박스. 100자 제한 및 글자 수 인디케이터 제공.

    *   `카테고리 그리드`: 6가지 카테고리가 2x3 또는 3x2 그리드 카드로 제공됨. 선택 시 강조 백그라운드가 켜짐.

    *   `내용 입력창`: 구체적인 작성을 유도하는 2000자 제한 텍스트에리어.

    *   `작성 꿀팁 박스`: 양질의 아이디어를 쓰기 위한 4가지 체크리스트 팁 박스 내장.

    *   `태그 입력 영역`: 사용자가 입력한 태그가 칩(tag-chip) 형태로 실시간 누적되는 인풋 컨테이너.

3.  **실시간 미리보기 패널 (우측/고정 영역)**

    *   `실시간 미리보기 카드`: 작성 중인 제목, 카테고리, 내용, 태그 정보가 실제 피드에 등록될 형태와 100% 동일한 미니어처 카드로 미러링 표시됨.

    *   `상단 펄스 이펙트`: '실시간 미리보기' 텍스트 옆에 붉은 지표등이 숨쉬듯 반짝이는 애니메이션 효과 제공.

    *   `좋은 아이디어 요약 가이드`: 피드백 퀄리티 제고를 위한 3가지 행동 요약 가이드 카드.

  

### 4.2. 화면 흐름 및 인터랙션

*   **동적 글자 수 경고**: 제목 80자 이상 시 경고색(노란색), 100자 도달 시 한계색(빨간색)으로 카운팅 배지가 변하여 즉각 통제합니다.

*   **인라인 태그 칩 처리**: 태그 작성 박스 클릭 시 인풋이 활성화되며 단어를 쓰고 Enter 혹은 쉼표(,) 입력 시, 단어가 자동으로 둥근 칩 배지 모양으로 바뀌고 닫기 버튼이 추가됩니다. 백스페이스(Backspace)를 누르면 가장 마지막 태그 칩이 삭제됩니다.

*   **실시간 유효성 검사**: 아래 세 조건을 동시에 만족하지 않으면 하단의 `아이디어 공유하기` 제출 버튼은 완전 비활성화(disabled) 처리됩니다.

    1.  제목 글자 수 > 5자

    2.  본문 글자 수 > 10자

    3.  카테고리 6개 중 반드시 1개 선택됨

*   **단축키 지원**: 마우스 클릭 없이 입력 도중 `Ctrl + Enter`를 누르면 즉시 작성 유효성을 확인하고 폼을 제출합니다.

  

### 4.3. 예외 처리 & 마이크로 UX

*   **태그 개수 초과 제한**: 태그를 5개 등록하면 더 이상 입력되지 않도록 텍스트 필드를 비활성화하고 플레이스홀더를 "최대 5개"로 강제 수정합니다.

## 5. 개발자 쇼케이스 페이지 (index.html - Showcase Tab)

  

*   **화면 목적**: 아이디어 피드를 거쳐 실제 개발이 성공적으로 완료된 결과물들을 카드형 디자인으로 탐색할 수 있는 수요자 중심의 갤러리 공간입니다.

  

### 5.1. 주요 UI 컴포넌트

1.  **헤더 섹션**: "개발자 쇼케이스", "Solve It의 아이디어로 탄생한 실제 서비스들을 만나보세요"라는 명확한 브랜딩 타이틀.

2.  **서비스 카드 그리드**

    *   `서비스 썸네일 영역`:

        *   서비스를 돋보이게 하는 대표 이미지 또는 아이콘 배치.

        *   우측 상단 투명 반투명 오버레이로 추천 획득수 노출.

    *   `서비스 본문 영역`:

        *   카테고리 배지 및 개발자 신뢰도 등급 랭킹 메달 표시(1등급, 2등급 등).

        *   서비스 이름 및 상세 내용 요약.

        *   활용 기술 및 기능이 포함된 해시태그 카드 제공.

    *   `쇼케이스 푸터`:

        *   **좌측**: `Original By @아이디어제안자닉네임` 표기로 기여자 크레딧 배치.

        *   **우측**: `방문하기` 아웃라인 버튼 (해당 서비스의 실제 주소로 연계).

  

### 5.2. 화면 흐름 및 인터랙션

*   **마이크로 리프트 오버 효과**: 마우스를 카드 위에 가져가면 카드가 브랜드 포인트 컬러(오렌지 광채)로 외곽선이 빛나며, 위로 3px 가량 부드럽게 상승(transform)하여 사용자의 적극적인 클릭을 유도합니다.

## 6. 공통 인증 모달 (Common Auth Modal)

  

*   **화면 목적**: 로그인 및 회원가입 처리를 통해 사용자가 개인화된 환경(내 정보 조회, 내가 쓴 아이디어 관리, 알림 등)을 활용할 수 있도록 지원하는 공통 팝업 창입니다.

  

### 6.1. 주요 UI 컴포넌트

1.  **모달 레이아웃**: 반투명 딤(Dim) 처리된 배경 패널(backdrop-filter: blur(5px)) 중앙에 모달 카드가 조명이 켜지듯 아래서 위로 부드럽게 나타납니다.

2.  **모달 상단 탭**: `로그인` / `회원가입` 스위칭 버튼 탭 구조.

3.  **로그인 폼 세부 컴포넌트**

    *   이메일/닉네임 입력란 및 패스워드 비밀 번호 입력창.

    *   `로그인` 버튼 (전체 너비형 브랜드 오렌지 채색 버튼).

    *   소셜 연동 영역: 구글 소셜 로그인 버튼, 카카오톡 소셜 로그인 버튼(카카오 브랜드 옐로우 테마 적용).

4.  **회원가입 폼 세부 컴포넌트**

    *   닉네임, 이메일, 패스워드 입력란.

    *   `역할 라디오 카드 그룹 (Role Selection)`:

        *   `제공자 카드`: "아이디어를 공유하고 싶어요" 설명 포함.

        *   `수요자 카드`: "개발 아이템을 찾고 있어요" 설명 포함.

        *   선택한 카드는 테두리와 배경색이 브랜드 오렌지 컬러로 동적 활성화됩니다.

  

### 6.2. 화면 흐름 및 인터랙션

*   **영역 외 클릭 시 닫힘**: 모달 카드 영역 밖의 어두운 반투명 오버레이를 클릭하면 모달창이 바로 사라지며, 작성 상태 등이 깨끗하게 초기화됩니다.

*   **로그인 성공 알림**: 로그인 완료 시 "로그인 됐어요!" 성공 토스트를 출력하며 자연스럽게 모달을 제거합니다.
